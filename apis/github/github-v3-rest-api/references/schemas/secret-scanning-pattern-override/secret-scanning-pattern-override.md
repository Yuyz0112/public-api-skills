# secret-scanning-pattern-override

**Type:** object

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `token_type` | string | No | The ID of the pattern. |
| `custom_pattern_version` | string | No | The version of this pattern if it's a custom pattern. |
| `slug` | string | No | The slug of the pattern. |
| `display_name` | string | No | The user-friendly name for the pattern. |
| `alert_total` | integer | No | The total number of alerts generated by this pattern. |
| `alert_total_percentage` | integer | No | The percentage of all alerts that this pattern represents, rounded to the nearest integer. |
| `false_positives` | integer | No | The number of false positive alerts generated by this pattern. |
| `false_positive_rate` | integer | No | The percentage of alerts from this pattern that are false positives, rounded to the nearest integer. |
| `bypass_rate` | integer | No | The percentage of blocks for this pattern that were bypassed, rounded to the nearest integer. |
| `default_setting` | enum: disabled, enabled | No | The default push protection setting for this pattern. |
| `enterprise_setting` | enum: not-set, disabled, enabled | No | The push protection setting for this pattern set at the enterprise level. Only present for partner patterns when the organization has a parent enterprise. |
| `setting` | enum: not-set, disabled, enabled | No | The current push protection setting for this pattern. If this is `not-set`, then it inherits either the enterprise setting if it exists or the default setting. |
